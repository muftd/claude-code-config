# Observer Pattern Recognition

## 认知偏差模式识别

### 1. 完美主义模式

#### 关键词识别
- **完美主义词汇**: 完美、最优、最佳、精确、确保每个、每一个细节
- **极端表达**: 必须、一定要、绝对不能、丝毫不能
- **重复强调**: 反复提及质量、标准、要求

#### 语义特征
- 时间投入与产出严重不成比例
- 过度关注细节而忽略整体进度
- 对错误或缺陷的过度恐惧

#### 情绪信号
- 焦虑感驱动完美主义行为
- 对不完美的强烈恐惧
- 控制欲强烈

### 2. 认知负荷转移模式

#### 关键词识别
- **多任务表达**: 同时处理、多个项目、并行、一边...一边...
- **信息过载**: 大量信息、太多选择、信息爆炸、处理不过来
- **进展缓慢**: 感觉缓慢、进展不大、效率低、卡住

#### 语义特征
- 注意力频繁切换
- 深度工作时间碎片化
- 决策疲劳明显

#### 情绪信号
- 压迫感和紧张感
- 思维混乱
- 焦虑和无力感

### 3. 系统性思维过载模式

#### 关键词识别
- **复杂化表达**: 综合分析、全面考虑、系统性、多维度
- **分析词汇**: 分析、研究、梳理、框架、模型
- **深度强调**: 深入、透彻、详细、全面

#### 语义特征
- 过度使用分析框架
- 分析深度与实际需求不匹配
- 理论分析脱离实践需求

#### 情绪信号
- 理性化倾向
- 对简单解决方案的不信任
- 对复杂性保护的依恋

## 复杂度评估算法

### 简单场景 (Skill直接处理)
- **单维度偏差**: 只检测到一种认知偏差模式
- **轻度表现**: 关键词出现频率低，表达温和
- **具体场景**: 针对特定任务或短期目标

### 复杂场景 (调度Sub Agent)
- **多维度偏差**: 同时检测到2-3种认知偏差模式
- **重度表现**: 关键词高频出现，表达强烈
- **系统性问题**: 涉及长期行为模式或工作方法论
- **专业领域需求**: 需要特定领域的专业知识 (AI PM、知识管理等)

## 观察策略库

### 轻量观察 (Skill)
```
完美主义: "很好奇，在[当前任务]中，'完美'和'完成'哪个更重要？"
认知负荷: "我注意到你在同时处理多个[事项]。很好奇，哪个是最关键的？"
系统思维: "听起来你在构建很全面的[模型]。很好奇，哪个是当前的瓶颈？"
```

### 深度分析建议 (调度Sub Agent)
```
检测到[偏差类型] + [附加因素]。这对[用户类型]很常见。
需要专业的认知建筑师帮你分析[专业领域]的策略吗？
```

## 个性化适配

### 根据用户背景调整
- **AI PM**: 关注prompt优化、产品迭代、工具选择
- **开发者**: 关注代码重构、架构设计、技术决策
- **知识工作者**: 关注学习策略、知识管理、分析方法

### 根据项目类型调整
- **创新项目**: 容忍一定模糊性，避免过度分析
- **工程项目**: 强调质量和标准，但需平衡进度
- **研究项目**: 鼓励深度思考，但需产出导向
