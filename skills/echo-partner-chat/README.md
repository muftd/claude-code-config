# Echo AI Partner - Claude Skill

## 🎯 概述

Echo AI Partner 是一个基于深度用户画像的个性化AI对话伙伴。通过分析你的AI对话记录构建个人画像，实现真正个性化的对话体验。

**特色**：
- 🧠 **深度记忆**：基于你的笔记和对话记录构建四层画像
- 🎭 **自然对话**：像朋友一样自然引用你的过往经历
- 📚 **持续学习**：从每次对话中学习，不断更新对你的理解
- 🚀 **开箱即用**：放入Claude Code即可使用，无需复杂配置

## 🛠️ 核心架构

### 配置文件

1. **user-persona.json** - 用户深度画像
   - 基于3060篇笔记的智谱API分析
   - 四层画像结构：身份、偏好、经历、洞察
   - 认知建筑师角色的完整刻画

2. **echo-persona.md** - Echo人格设定
   - 温暖理解的沟通风格
   - 自然引用原则
   - 记忆温度和个性化支持

### 工作原理

```
用户对话 → Claude加载Skill → 读取配置文件 → 个性化回应
    ↓           ↓                ↓              ↓
  问题输入   → echo-partner-chat → 画像+人设 → Echo回应
```

## 🔄 更新机制

### 自动更新
- 每次对话后自动分析新的偏好和行为模式
- 使用Read-Modify-Write策略更新画像
- 强化重要经历和偏好的记忆权重

### 手动更新
```bash
# 基于新笔记更新画像
python scripts/echo_initializer.py \
  --notes-dir /path/to/new/notes \
  --output-dir ./config
```

## 🚨 故障排除

### 常见问题

**1. API调用失败**
```
错误: ZHIPU_API_KEY not found
解决: export ZHIPU_API_KEY="your-key"
```

**2. 依赖包缺失**
```bash
错误: ModuleNotFoundError: No module named 'chromadb'
解决: pip install -r scripts/requirements.txt
```

**3. 向量数据库创建失败**
```
错误: Permission denied when creating vector_db
解决: 确保输出目录有写入权限
```

### 调试模式
```bash
# 启用详细日志
python scripts/echo_initializer.py --notes-dir /path/to/notes --tag "啊AI" 2>&1 | tee echo_debug.log
```
